# pyvelm v0.6.0

**Release date:** 2026-05-23

Visual approval workflows — design state machines, run them on any model, and
sign off from the form bar or a dedicated inbox.

---

## Highlights

### Workflow module

Install **Apps → Workflows** (module version **0.2.0**, depends on `base` +
`admin`).

- **Designer** — `/web/workflow/build`: states, transitions, approval rules,
  stage forms (new fields or writable record fields), auto-start on create.
- **Runtime bar** — on record forms: pipeline status bar, start/transition
  buttons, pending approve/reject when assigned to you.
- **My approvals** — `/web/workflow/inbox` lists every pending sign-off;
  approve, reject, or open the underlying record.
- **Tasks** — assigned approvers get `workflow.task` rows.
- **Escalation** — cron job every 15 minutes for overdue approvals (install/sync
  seeds it).

Approval strategies: **any** (one approver), **all** (every group member), or
**sequential** (one after another). Optional deadline hours and escalate-to-group.

All workflow dialogs use the draggable **`PvDialog`** component (`pvConfirm` for
simple confirms; HTMX-loaded stage forms for transition fields).

### Examples

- **Partners** — partner onboarding workflow when both `partners` and `workflow`
  are installed (chatter + workflow state column).
- **Feedback Signals** — “Feedback intake review” workflow seeded on sync when
  the workflow module is present.

See [Workflows](../workflow.md) for the full guide.

## Upgrade

```bash
pip install -U pyvelm
```

1. **Apps → Workflows → Install** (or Sync if already installed).
2. Optional: **Apps → Partners** / **Feedback Signals → Sync** for example
   workflow definitions.
3. Open **Admin → Workflows → Design workflow** or **My approvals** from the
   menu.

No new PyPI dependencies beyond v0.5.0.
