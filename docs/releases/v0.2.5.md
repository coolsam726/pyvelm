# pyvelm v0.2.5

**Release date:** 2026-05-22  
**Base module:** `0.18.0` (unchanged)

Artisan console, focused code generators, dev/production server modes,
and a fix for empty registries under uvicorn reload.

---

## Highlights

### Artisan console

```bash
pyvelm make:module inventory
pyvelm make:model inventory.product --module=inventory
pyvelm make:view inventory.product --module=inventory
pyvelm make:menu --view=product.list --module=inventory
pyvelm db autogen inventory --with-views
```

See [Console commands](../console.md).

### Dev vs production

```bash
PYVELM_ENV=development python -m app.serve --reload   # /docs, debug logs
PYVELM_ENV=production  gunicorn -c gunicorn_conf.py app.serve:app
```

Docker: `docker compose -f docker-compose.yml -f docker-compose.dev.yml up`

### Reload fix

`uvicorn --reload` no longer leaves a fresh `Registry` without models
(login “User model not loaded”). The loader re-syncs model classes from
cached Python modules after each import.

---

## Install

```bash
pip install pyvelm==0.2.5
```

For local development on this repo:

```bash
pip install -e .
```

---

## Upgrade from v0.2.4

1. `pip install -U pyvelm==0.2.5` (or `pip install -e .` from source)
2. Restart the app process
3. Regenerate modules with `make:*` if you still have old sample stubs
