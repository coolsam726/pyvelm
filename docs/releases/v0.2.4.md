# pyvelm v0.2.4

**Release date:** 2026-05-22  
**Base module:** `0.18.0` (unchanged)

Menus builder, CLI module-root auto-discovery, and a sidebar icon fix.

---

## Highlights

### `Menus` builder

Stop hand-building parent keys and view URLs:

```python
from pyvelm.builders import Menus

m = Menus("partners")  # NAME in __pyvelm__.py

MENUS = [
    m.group("business", "Business", sequence=50),
    m.item("business.partners", "Partners",
           parent="business", view="partner.list", sequence=10),
    m.item("business.tags", "Tags",
           parent=("admin", "settings"), view="tag.list", sequence=40),
]
```

| You write | Resolved |
|-----------|----------|
| `parent="business"` | `partners.business` |
| `view="partner.list"` | `/web/views/partners/partner.list` |

### CLI module discovery

`pyvelm cron` and `pyvelm db` now read `modules_root` from `pyvelm.toml`
in cwd or a parent — no need to set `PYVELM_MODULE_ROOTS` for local dev.

### Sidebar

Menu groups without an `icon` no longer show the text `None`.

---

## Install

```bash
pip install pyvelm==0.2.4
```

---

## Upgrade from v0.2.3

1. `pip install -U pyvelm==0.2.4`
2. Optionally refactor `views/menu.py` to use `Menus` (old `menu_item` still works)
3. Restart app / cron
